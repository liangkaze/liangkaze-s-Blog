# 运行ansible的基本认识和前期准备

## Ansible基本认识

### 概述

Ansible 是一款**开源 IT 自动化运维工具**，基于 Python 开发，由 Red Hat 维护。
核心优势：**无代理、基于 SSH、YAML 语法、幂等性**，广泛用于批量运维、配置管理、应用部署、编排调度。

---

###  核心特点

- **无代理（Agentless）**
  被控端无需安装客户端，仅需 SSH + Python 即可。
- **简单易用**
  任务用 **YAML** 编写，结构清晰，接近自然语言。
- **幂等性**
  多次执行同一任务结果一致，不会重复操作。
- **模块化**
  内置大量模块（文件、服务、包管理、用户、云等）。
- **跨平台**
  支持 Linux、Windows、网络设备、云主机等。

---

### 核心组件

####  控制节点 & 被控节点

- **控制节点**：安装 Ansible，执行命令/剧本（Linux/macOS）。
- **被控节点**：被管理机器，无需安装 Ansible。

####  Inventory（主机清单）

定义要管理的主机与分组，支持 INI/YAML 格式。

```ini
[web]
192.168.1.10
192.168.1.11

[db]
192.168.1.20
```

#### Module（模块）

完成具体操作的最小单元，如 `ping`、`yum`、`copy`、`service`、`user`。

#### Ad-Hoc 临时命令

快速执行单条任务，适合临时操作。

```bash
ansible all -m ping
ansible web -m command -a "uptime"
```

#### Playbook（剧本）

YAML 编写的自动化任务集合，可批量、有序执行。

```yaml
- name: 安装并启动 nginx
  hosts: web
  tasks:
    - name: install nginx
      yum:
        name: nginx
        state: present

    - name: start nginx
      service:
        name: nginx
        state: started
        enabled: yes
```

#### Role（角色）

对 Playbook 进行结构化封装，实现复用（如 nginx、mysql、docker 角色）。

#### Ansible Vault

加密密码、密钥等敏感信息。

---

### 工作流程

1. 控制节点读取 Inventory 和 Playbook
2. 通过 SSH 连接被控端
3. 推送并执行模块
4. 返回结果并清理临时文件

---

### 典型应用场景

- 批量执行命令、批量修改配置
- 软件统一安装、服务启停管理
- 自动化部署应用、CI/CD 集成
- 云资源、网络设备自动化管理
- 系统安全加固与合规巡检

---

### 同类工具对比

- **Ansible**：无代理、轻量、上手快、YAML
- **SaltStack**：速度快，支持代理/无代理
- **Puppet/Chef**：偏重，适合大规模复杂环境

## 为三台虚拟机创建用户修改主机名

```shell
#修改主机名为你的姓名简拼+01/02/03
```

```shell
#在三台虚拟机上创建三同名用户
[root@example01 /]# useradd example
```

## 免密登录

首先三台虚拟机在/etc/hosts 中修改目标IP的主机名做一个本地DNS

```shell
[root@example01 /]# vim /etc/hosts
# Loopback entries; do not change.
# For historical reasons, localhost precedes localhost.localdomain:
127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
# See hosts(5) for proper format and other examples:
# 192.168.1.10 foo.example.org foo
# 192.168.1.13 bar.example.org bar
 192.168.177.129 example01
 192.168.177.130 example02
 192.168.177.131 example03                                  
```

开始生成SSH证书密钥

```shell
[root@example01 /]# ssh-keygen -t rsa
[root@example01 /]# ssh-copy-id example@example01
[root@example01 /]# ssh example@openeuler01
```

可以免密登录
